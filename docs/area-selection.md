# 地域選択の仕様

このドキュメントは、UIにおける地域選択の仕様と、JMAの内部階層との対応について解説する。  
開発者およびAIエージェントが混乱しないよう、設計上の意図も合わせて記載する。

---

## UIの地域選択階層

UIでは地域を以下の **3段階** で選択する。

| ステップ | UI上の表現 | JMA内部の対応 |
| -------- | ---------- | ------------- |
| 1        | 地方       | center        |
| 2        | 都道府県   | office        |
| 3        | 市区町村   | class20       |

> ⚠️ UIには **class10（一次細分区域）** および **class15（二次細分区域）** を **表示しない**。

---

## JMAの階層構造との対比

気象庁のAPIが返す地域データは5段階の階層を持つ。

```
center（地方予報区）
  └─ office（府県予報区）
       └─ class10（一次細分区域）
            └─ class15（二次細分区域）
                 └─ class20（市区町村等をまとめた地域）
```

UIはこのうち **center・office・class20** の3段階のみ露出し、**class10・class15はUIに出さない**。

---

## 設計の理由

### class10・class15を省略する理由

- class10・class15は「南部」「北東部」「沿岸部」のような **曖昧な地理表現** が多い
- ユーザーが目的の市区町村（class20）がどの区域に属するか判断しづらく、選択操作のハードルが上がる

### 地方・都道府県を残す理由

- 候補の **絞り込み** としてユーザーに有効
- 都道府県単位はユーザーの日常認識と一致しており、直感的に選択できる

### 最小構成との比較

- システム的には class20 コードだけで天気予報の取得は成立する
- ただし、UX上の **探索性（discoverability）** を確保するために地方・都道府県の2段階を残している

---

## 内部実装との関係

class10・class15はUIには表示されないが、内部では引き続き利用される。

- `SelectedArea` 型（`src/lib/jma/selection/types.ts`）は全5階層を保持する
- `generateAreaOptions`（`src/lib/jma/selection/utils.ts`）は全階層の選択肢を生成する
- UIコンポーネントは center・office・class20 の選択肢のみをユーザーに提示する

---

## まとめ

| 項目                     | 内容                                     |
| ------------------------ | ---------------------------------------- |
| UIの選択ステップ数       | 3段階（地方 → 都道府県 → 市区町村）      |
| UIに表示しない階層       | class10（一次細分区域）・class15（二次細分区域） |
| 内部で保持する階層       | 全5階層（center / office / class10 / class15 / class20） |
| 天気予報取得に使うコード | class20 コード                           |
