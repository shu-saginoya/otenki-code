# コンポーネント設計書

このドキュメントは開発者およびAIエージェント向けに、本プロジェクトのディレクトリ構成について解説する。

## ディレクトリ構成の概要

- app # Next.jsのルーティングシステム
- components # 再利用可能なコンポーネント
  - features # アプリケーション固有の機能コンポーネント
  - ui # 汎用的なUIコンポーネント
- data # テキスト・画像のパスなどコンテンツ
- hooks # 再利用可能なロジック
- lib # グローバルステートや共通ライブラリなど
- styles # CSS関連のユーティリティー関数
- types # 共通の型定義
- utils # 共通のユーティリティ関数

---

## 📂 `app/`

Next.jsのApp Routerに対応したディレクトリ。
命名や校正などはNext.jsの公式に準拠する。

- 例: `page.tsx`, `layout.tsx`

## 📂 `components/`

再利用可能なコンポーネントを配置する。

- 命名はPascalCase
- Storybookを設置する

### 📂 `components/features/`

アプリケーション固有の機能を実装するコンポーネントを配置。

- `area/`: エリア選択関連のコンポーネント
- `clothing/`: 服装推薦関連のコンポーネント
- `common/`: アプリ全体で使用される共通コンポーネント
- `forecast`: 天気予報関連のコンポーネント

### 📂 `components/ui/`

アプリケーション非依存の汎用的なUIコンポーネントを配置。
他のプロジェクトでも流用できるよう、外部依存を最小限にする。

- `buttons/`: ボタン関連のコンポーネント
- `display/`: 表示関連のコンポーネント
- `inputs/`: 入力要素関連のコンポーネント
- `layout/`: レイアウト関連のコンポーネント

## 📂 `data/`

テキストや画像へのパスなどコンテンツを格納する

- 命名はcamelCase

## 📂 `hooks/`

再利用可能なロジックを定義する。

- ReactやNext.jsの機能に依存している関数
- 命名はcamelCase
- 命名は`useXXXXX`のように`use`プレフィックスをつける

## 📂 `lib/`

グローバルステートや共通ライブラリを格納する。

- 例: `cn.ts`, `dayjs.ts`, `hooks.ts`, `store.ts`など

## 📂 `styles/`

CSS関連のユーティリティー関数を格納。

- `parts/`: 基本的なスタイルユーティリティ
- `templates/`: テーマやバリエーション

## 📂 `types/`

プロジェクト全体で利用する型定義ファイルを格納。

## 📂 `utils/`

プロジェクト全体で使うユーティリティ関数を格納。

- Reactに依存しない
- 純粋関数

## 🚀 運用ルール

- `types/`でTypeScriptの型定義を管理し、開発の一貫性を保つ
- 原則、`hooks/` = React依存、`utils/` = 純粋関数
